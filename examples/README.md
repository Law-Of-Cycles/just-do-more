# Worked Comparisons

These examples compare a baseline answer with a Just Do More style answer for the same prompt.

Use them to study the mechanism, especially these recurring shifts:

| Baseline tendency | Just Do More tendency |
|---|---|
| answers the surface wording | addresses the likely decision underneath it |
| uses the default template for the question type | adjusts to the actual constraint |
| stays balanced and generic | becomes more direct and situational |
| covers many expected points | prioritizes the one or two points that matter most |

## The cases

| Case | Prompt | Main shift |
|---|---|---|
| [1 — Framework Choice](case-1-framework-choice.md) | Should I learn React or Vue first? | moves from comparison to action guidance |
| [2 — Team Efficiency](case-2-team-efficiency.md) | How do I improve my team's efficiency? | moves from advice to diagnosis |
| [3 — Career Crossroads](case-3-career-crossroads.md) | I feel lost after three years here. Should I switch jobs? | separates location problems from identity problems |
| [4 — Salary Negotiation](case-4-salary-negotiation.md) | How should I negotiate a raise? | focuses on leverage before scripts |
| [5 — Mobile Strategy](case-5-mobile-strategy.md) | Should my SaaS build a mobile app? | replaces broad pros and cons with forcing conditions |
